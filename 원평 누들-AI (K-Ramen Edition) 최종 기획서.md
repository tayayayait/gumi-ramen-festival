# **\[프로젝트 개요\]**

* **프로그램명**: 원평 누들-AI (Wonpyeong Noodle-AI: K-Ramen Edition)  
* **핵심 컨셉**: 구미 농심 공장의 '갓 튀긴 라면'을 베이스로, AI가 지역 특산물 토핑 조합을 제안(B2B)하고 사용자가 AR로 재료를 수집(B2C)하는 **하이테크 로컬 상권 활성화 플랫폼**  
* **주요 타겟**: 원평동 누들로드 입점 소상공인 및 구미시 방문 관광객(MZ세대 및 가족 단위)  
* **지원 사업 연계**: 2026년 구미시 청년창업지원사업 가점 항목인 **'원평동 구도심 일원 사업장'** 및 **'6차 산업(농업 연계)'** 조건을 동시 충족하여 선발 가능성 극대화 1

# **\[프로젝트 배경 및 목표\]**

* **구미시 정책 부합성**:  
  * **원평동 상권 활성화**: 2030년까지 100억 원이 투입되는 '문화로 자율상권 활성화 사업'과 연계하여 누들로드 특화골목 조성 지원 2  
  * **라면 축제 브랜드화**: 구미 라면축제의 '갓 튀긴 라면'과 '이색 레시피' 콘텐츠를 상설화하여 축제 외 기간에도 유동인구 유지  
  * **푸드테크 거점 활용**: 선산읍 푸드테크 연구지원센터(282억 원 규모)의 AI 및 스마트 제조 실증 인프라를 상권 데이터와 연결 3  
* **목표**:  
  * 소상공인의 낮은 디지털 기술 활용도(15.4%)를 개선하는 스마트 오더 및 마케팅 툴 보급 5  
  * AR 게이미피케이션을 통한 구도심 체류 시간 및 재방문율 증대

# **상세 기능 요구 사항**

### **1\. B2B: 상인용 'AI 라면 커스텀 어드바이저'**

* **AI 레시피 생성**: Gemini API를 활용, 신라면·짜파게티 등 기성 라면에 구미한우, 장천 멜론 소스 등 지역 농산물 토핑을 조합한 이색 조리법 제안  
* **스마트 주문 및 공급망**: 축제에서 검증된 QR 주문 시스템을 도입하고, 푸드테크 센터의 식자재 데이터와 연동하여 재고 최적화 지원  
* **타겟 마케팅**: 유동인구 분석 API를 통해 상권이 한산한 시간에 맞춰 근처 사용자에게 타임 세일 쿠폰 자동 발행

### **2\. B2C: 고객용 '나만의 라면 탐험 (AR Scavenger Hunt)'**

* **AR 재료 수집 게임**: ARCore Geospatial API를 활용해 문화로 거리 곳곳에 배치된 가상 라면 스프, 계란, 파 아이템을 수집  
* **보상 시스템**: 특정 재료 세트를 완성하면 누들로드 매장에서 즉시 사용 가능한 토핑 추가권 또는 G-코인(지역 포인트) 지급  
* **소셜 레시피 공유**: 사용자가 만든 커스텀 라면 사진을 공유하고 인기 레시피를 매장에 제안하는 '라면 공작소' 커뮤니티

# **핵심 기술 스택 및 API 상세**

| 구분 | 기술 / API 명칭 | 주요 역할 및 상세 기능 |
| :---- | :---- | :---- |
| **AR 엔진** | **Google ARCore Geospatial API** | 구글 스트리트 뷰 기반 VPS 기술로 실제 건물 위치에 AR 재료를 센티미터 단위로 정밀 배치 |
| **3D 공간 설계** | **Google Map Tiles API** | Unity 에디터 내에서 구미 원평동의 3D 포토리얼리스틱 지도를 불러와 아이템 위치를 입체적으로 설계 |
| **생성형 AI** | **Google Gemini API** | 사용자의 입맛 취향과 구미 6차 산업 농산물 데이터를 분석하여 맞춤형 커스텀 라면 레시피 생성 |
| **공간 DB** | **Supabase PostGIS** | 유저의 현재 위치와 AR 아이템 간의 거리를 계산(st\_dwithin)하여 수집 가능 여부를 판단 |
| **서버 로직** | **Supabase Edge Functions** | 보안이 강화된 서버리스 환경에서 Gemini API 호출 및 쿠폰 발급 로직을 처리 |
| **보안/인증** | **Supabase Auth & RLS** | 사용자 세션 관리 및 Row Level Security를 통한 개인 정보 및 포인트 데이터 보호 |

# **비기능적 요구사항**

* **보안 및 부정행위 방지**:  
  * GPS 조작(Spoofing)을 방지하기 위해 서버 측에서 가속도 센서 데이터와 위치 이동 속도의 타당성을 실시간 검증  
  * Edge Functions에서 API 키를 암호화하여 관리하고 외부 노출 차단  
* **성능 최적화**: 저사양 기기에서도 원활한 AR 구동을 위해 가시거리 밖의 메시(Mesh)를 제거하는 최적화 알고리즘 적용  
* **접근성**: 고령 상인들을 고려하여 텍스트 위주가 아닌 '이미지'와 '음성 명령' 중심의 직관적인 UI 설계

# **\[지원 디바이스 및 디자인\]**

* **지원 디바이스**: Unity AR Foundation 기반 Android(ARCore) 및 iOS(ARKit) 동시 지원  
* **디자인 컨셉**:  
  * 구미의 하이테크 이미지를 상징하는 '사이버 펑크 블루'와 라면의 역동성을 상징하는 '네온 오렌지'의 조화  
  * MZ세대의 인스타그램 감성을 자극하는 세련된 AR 캐릭터 및 아이템 디자인
